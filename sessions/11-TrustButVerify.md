# Session 11 - Trust, But Verify: Giving Your AI Agents a Safety Valve

- **Speakers**: [Wycliffe Maina](https://www.linkedin.com/in/mainawycliffe/)

## About the session
As applications evolve from simple chatbots to autonomous agents, engineering deterministic safety boundaries becomes critical. This session explores the architectural blueprint of Human-in-the-Loop workflows using Firebase Genkit and Angular. We will look at how to design intelligent systems that know exactly when to pause themselves when high-risk actions are triggered, preventing agents from going rogue. You will discover how a modern client-server architecture can cleanly bridge these paused backend states with a reactive frontend, transforming abstract AI decisions into clear, interactive approval interfaces that keep the user firmly in control.

Session Goals

- Understand the Interrupt Paradigm: Learn how to design system boundaries that safely halt executing agent flows before critical operations commit.
- Bridge AI State to User Experience: Discover how to transition a web interface fluidly between open-ended conversational streams and structured validation states.
- Enforce UI-Driven Security Gates: Explore why discrete, event-driven web components are the most effective boundary for preventing model hallucination and unauthorized action execution.
